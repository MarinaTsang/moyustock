# 股票行情插件 - 产品需求文档（PRD）

## 文档信息

- **产品名称**：股票行情（摸鱼看盘）
- **文档版本**：v1.0
- **创建日期**：2026-02-03
- **目标受众**：开发团队、产品团队

## 1. 产品概述

### 1.1 产品定位

一款面向上班族的Chrome浏览器插件，在工作场景下低调、便捷地查看股票行情，实现"摸鱼看盘"的核心需求。

### 1.2 目标用户

- **主要用户**：需要在工作时间查看股票行情的上班族
- **用户特征**：
  - 使用Chrome浏览器
  - 需要实时了解股票价格变化
  - 希望界面低调，不引起注意
  - 对操作便捷性要求高

### 1.3 核心价值

- **低调**：小巧的浮窗设计，不占用过多屏幕空间
- **便捷**：常驻显示，无需频繁打开应用
- **实时**：自动刷新数据，保持信息最新
- **智能**：自动识别股票代码，支持多股票轮播

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 浮窗显示
- **需求**：在网页右下角显示一个可拖拽的浮窗
- **优先级**：P0（必须）
- **详细说明**：
  - 浮窗默认位置：浏览器右下角，距离边缘24px
  - 浮窗尺寸：宽度188px，高度自适应（最多3行）
  - 支持拖拽移动，位置自动保存
  - 支持关闭/重新打开

#### 2.1.2 股票行情显示
- **需求**：实时显示股票名称、价格、涨跌幅
- **优先级**：P0（必须）
- **详细说明**：
  - 显示格式：`股票名称 价格 涨跌幅%`
  - 涨跌颜色：红色=涨，绿色=跌
  - 最多显示3只股票
  - 超过3只时自动轮播

#### 2.1.3 大盘指数显示
- **需求**：在浮窗标题栏显示上证指数
- **优先级**：P0（必须）
- **详细说明**：
  - 显示格式：`指数点数 涨跌幅%`
  - 不显示"上证指数"文字
  - 颜色跟随涨跌变化
  - 独立于股票列表，不参与轮播

#### 2.1.6 分时走势线（新）
- **需求**：在股票价格下方显示当日简易分时走势
- **优先级**：P1（重要）
- **详细说明**：
  - 位置：位于股票名称和价格下方
  - 样式：极细线条，无坐标轴
  - 颜色：
    - 涨（Change > 0）：淡红色
    - 跌（Change < 0）：淡绿色
    - 平（Change = 0）：灰色
  - 数据源：利用接口返回的简易分时数据绘制

#### 2.1.4 股票管理
- **需求**：支持添加、删除股票
- **优先级**：P0（必须）
- **详细说明**：
  - 添加：输入股票代码，自动识别交易所并补全前缀
  - 删除：在设置列表中删除
  - 数据持久化：使用chrome.storage.local存储
  - 默认股票：sh600519（贵州茅台）

#### 2.1.5 智能轮播
- **需求**：超过3只股票时自动轮播
- **优先级**：P1（重要）
- **详细说明**：
  - 轮播间隔：4秒
  - 轮播方式：向上滑动切换
  - 循环逻辑：使用 `(offset + i) % length` 实现无限循环
  - 悬停暂停：鼠标悬停时暂停轮播

### 2.2 辅助功能

#### 2.2.1 网络状态检测
- **需求**：检测网络状态，离线时显示提示
- **优先级**：P1（重要）
- **详细说明**：
  - 初始化时检查 navigator.onLine
  - 监听 online/offline 事件
  - 离线时显示"离线"状态
  - 重连后自动恢复数据获取

#### 2.2.2 请求超时处理
- **需求**：请求超时（3秒）时显示错误提示
- **优先级**：P1（重要）
- **详细说明**：
  - 使用 Promise.race 实现超时控制
  - 超时后保留旧数据，显示"数据可能过时"
  - 不停止定时器，继续尝试获取数据

#### 2.2.3 智能唤醒
- **需求**：切换标签页或唤醒屏幕时立即刷新数据
- **优先级**：P2（一般）
- **详细说明**：
  - 监听 document.visibilitychange 事件
  - visibilityState === 'visible' 时触发刷新
  - 不重建DOM，只更新数据

#### 2.2.4 数据缓存
- **需求**：请求失败时保留上次成功的数据
- **优先级**：P1（重要）
- **详细说明**：
  - 缓存大盘指数数据
  - 缓存各股票数据
  - 失败时显示缓存数据 + 过时提示

### 2.3 交互需求

#### 2.3.1 浮窗交互
- **拖拽**：点击标题栏拖拽移动
- **关闭**：点击×按钮关闭
- **重新打开**：点击插件图标唤醒（如果浮窗已关闭）
- **悬停暂停**：鼠标悬停时暂停轮播

#### 2.3.2 设置面板
- **切换**：点击浮窗中的⚙️按钮切换设置面板
- **添加股票**：在设置面板输入代码 + 点击添加按钮
- **删除股票**：在设置面板点击删除按钮
- **保存返回**：点击"保存"按钮返回行情显示（设置面板中的按钮）

## 3. 技术需求

### 3.1 数据源

- **接口地址**：`http://qt.gtimg.cn/q={code}`
- **编码格式**：GBK
- **数据格式**：使用`~`分隔的字符串
- **数据字段**：
  - 第1项：股票名称
  - 第3项：当前价格
  - 第32项：涨跌幅
  - 简易分时数据（待确认字段位置）

### 3.2 存储方案

- **存储方式**：chrome.storage.local
- **存储内容**：
  - stockList：股票代码数组
  - 浮窗位置（localStorage）
  - 浮窗关闭状态（localStorage）

### 3.3 性能要求

- **刷新频率**：每2秒刷新一次股票数据
- **轮播间隔**：4秒切换一次
- **请求超时**：3秒
- **DOM更新**：使用增量更新，避免闪烁

### 3.4 兼容性要求

- **浏览器**：Chrome浏览器（Manifest V3）
- **页面类型**：http:// 和 https:// 协议的网页
- **注入时机**：document_end

## 4. UI/UX需求

### 4.1 设计原则

- **低调**：浅灰色背景，不引人注目
- **简洁**：信息密度适中，易于阅读
- **紧凑**：占用空间小，不影响工作

### 4.2 视觉规范

- **字体**：系统默认字体，12px
- **颜色**：
  - 涨：`#c0392b`（红色）
  - 跌：`#27ae60`（绿色）
  - 中性：`#666`（灰色）
- **间距**：4px行间距，6-8px内边距
- **圆角**：4-8px

### 4.3 交互反馈

- **加载状态**：显示"加载中…"
- **错误状态**：显示错误信息或"网络重连中…"
- **过时状态**：显示"数据可能过时"
- **悬停效果**：按钮背景色变化

## 5. 异常处理

### 5.1 网络异常

- **离线**：显示"离线"状态，不发起请求
- **超时**：3秒超时，显示"数据可能过时"，保留旧数据
- **重连**：自动恢复数据获取

### 5.2 数据异常

- **格式错误**：显示"数据格式异常"
- **缺失字段**：使用默认值"—"显示
- **解析失败**：显示"加载失败"

### 5.3 用户操作异常

- **重复添加**：提示"该股票已存在"
- **空输入**：提示"请输入股票代码"
- **格式错误**：提示"股票代码格式不正确"

### 10.1 股票代码识别规则

| 交易所/板块 | 代码特征 | 前缀 | 示例 |
|------------|---------|------|------|
| 上海主板/科创板 | 6开头(6位) | sh | 600519 → sh600519 |
| 上海B股 | 9开头(6位) | sh | 900901 → sh900901 |
| 深圳主板/创业板 | 0或3开头(6位) | sz | 000001 → sz000001 |
| 深圳B股 | 2开头(6位) | sz | 200055 → sz200055 |
| 北交所 | 8或4开头(6位) | bj | 839719 → bj839719 |
| 港股 | 5位数字 | hk | 00700 → hk00700 |
| ETF/LOF基金 | 1开头(6位) | sz | 159875 → sz159875 |
| ETF/LOF基金 | 5开头(6位) | sh | 513301 → sh513301 |

### 10.2 数据接口说明

**请求示例**：
```
GET http://qt.gtimg.cn/q=sh600519
```

**响应格式**：
```
v_sh600519="1~贵州茅台~600519~1474.92~...~+3.36";
```

**解析方式**：
- 使用`~`分隔字符串
- 提取第1项（名称）、第3项（价格）、第32项（涨跌幅）
