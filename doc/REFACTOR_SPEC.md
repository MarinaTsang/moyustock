# 摸鱼看盘重构规格（REFACTOR_SPEC）

## 1. 产品目标
- 在不打断工作流的前提下，稳定地感知持仓/自选波动。
- 不错过关键波动，但避免重复打扰和系统通知式干扰。
- 视觉上“低存在感 + 高可恢复性”，长期挂在页面右下角。

## 2. 用户与场景
- 典型用户：多年上班族、盘中偶尔瞥一眼行情的股民。
- 主要场景：
  - 工作中后台挂窗，偶尔看一眼。
  - 关键波动出现时希望被轻提醒，但不希望反复提醒。
  - 盘后回看当日走势，不希望曲线“假波动”。

## 3. 功能范围（重构后）
1. 自选股管理：增删、跨标签同步。
2. 行情刷新：保留现有请求链路（`GET_STOCK`），不改数据源。
3. 轮播展示：>3 只时平滑轮播，无空白帧、无闪屏。
4. 关键态检测：阈值 + 持续时间 + 方向切换重计时。
5. 去重提醒：
   - 同股票同一波动只提醒一次；
   - 5 分钟冷却；
   - hover 已读后当日不再提醒。
6. 分时趋势线：
   - 语义为“当日交易时段价格点连线”；
   - 非交易时段不新增点，曲线保持静态可回看。
7. 开发调试面板（仅开发模式）：
   - 每只股票显示 critical/read/cooldown/sustain 等状态。

## 4. 非功能要求
- Manifest V3 兼容。
- 不修改行情请求逻辑与权限边界。
- 页面切换、休眠恢复、网络抖动后可恢复。
- 保持低 CPU/低重绘频率，避免明显卡顿。

## 5. 当前已知问题（重构前）
1. 轮播阶段偶发空白帧（刷新与动画争用 DOM）。
2. 趋势线语义偏差（未按当日交易时间轴建模）。
3. 关键态、轮播、渲染逻辑耦合在 `content.js`，维护成本高。

## 6. 分阶段实施

### Phase 1（稳定性优先）
- 目标：先消除闪屏，保证轮播与刷新稳定协同。
- 实施：
  - 增加轮播动画互斥状态，避免刷新重建 DOM 抢占动画。
  - 动画换页行先用缓存行情预填，减少占位闪现。
  - 异步回填支持“旧/新行 ID 双路径”，避免动画后找不到节点。
- 验收：
  - 连续运行 2 小时无空白帧；
  - 轮播期间无明显“加载中”闪烁。

### Phase 2（分时语义）
- 目标：趋势线变成“今日分时”而非短期抖动线。
- 实施：
  - 按市场交易时段建立时间轴（含 A/H 午休断档累计）。
  - 仅交易时段追加点；盘后保持静态曲线。
  - 按市场交易日切分日内点集。
- 验收：
  - 盘中曲线随交易推进；
  - 盘后不再波动；
  - 切页后曲线语义不丢失。

### Phase 3（状态机与可维护性）
- 目标：拆分职责，减少 `content.js` 耦合。
- 实施：
  - `anomalyMonitor` 负责检测与提醒决策；
  - `view model` 负责“当前展示哪些股票”；
  - `render layer` 仅负责最小 DOM 更新。
- 验收：
  - 核心逻辑可单测；
  - 关键时序回归通过。

## 7. 事件与状态模型（目标）
- 状态：`SILENT | ACTIVE | CRITICAL`
- 事件：
  - `BREACH_START`
  - `BREACH_STABLE`（达到持续时间）
  - `ALERT_EMIT`
  - `ALERT_SUPPRESS_COOLDOWN`
  - `ALERT_SUPPRESS_READ`
  - `BREACH_END`

## 8. 回归测试清单
1. 轮播稳定：
   - 3/4/10 只股票分别测试；
   - 网络慢速条件下连续轮播 30 分钟。
2. 提醒去重：
   - 同波动不重复；
   - 5 分钟冷却生效；
   - hover 当日已读生效。
3. 交易时段：
   - 盘中、午休、盘后三段行为正确。
4. 恢复能力：
   - 标签切换、休眠唤醒、断网重连后状态一致。

## 9. 当前进度（本轮）
- 已进入 Phase 1 + Phase 2 初步实现：
  - 轮播与刷新互斥；
  - 轮播行缓存预填 + 回填兜底；
  - 分时线按交易时段累计分钟建模并按交易日切分。
