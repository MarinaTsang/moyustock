## 产品升级与差异化方案（规划稿）

### 1. 产品定位与目标

- **产品一句话**：一款为上班族设计的「低打扰自选股雷达」，在不打断工作的前提下，帮你在关键时刻多看一眼行情。
- **核心目标**：
  - 默认安静，不主动打扰；
  - 有行情但不剧烈时，轻度存在感，给人“心里有数”的感觉；
  - 波动达到关键阈值且持续一段时间时，存在感明显但仍不打断。

### 2. 当前版本能力（Baseline）

- 常驻右下角浮窗，支持拖拽与位置记忆；
- 支持 A 股 / 港股代码自动补前缀，自选股列表持久化；
- 最多展示 3 行，超出后按 `(offset + i) % length` 轮播；
- 顶部显示上证指数（sh000001）点数与涨跌幅；
- Content Script 浮窗 + Background 代理行情请求（GBK 解码）；
- 已实现：
  - **定高循环展示**（>3 只时固定高度，循环补位，避免高度抖动）；
  - **无闪烁更新**（分离 DOM 创建与数据更新）；
  - **向上顶式轮播动画**；
  - **存在感三态（SILENT / ACTIVE / CRITICAL）**基础逻辑；
  - 静默态降频刷新（降低视觉打扰与网络压力）。

### 3. 差异化设计方向

1. **存在感三态 + 工作场景感知**
   - 静默：当前无在交易时间的股票，或波动极小 → 浮窗透明、偏灰，不轮播或低频刷新；
   - 活跃：有在交易时间的股票，但波动 < ±3% → 正常轮播与刷新；
   - 关键：至少一只在交易时间、涨跌幅 ≥ ±3%，且持续 ≥ N 秒 → 浮窗高亮、边框强调，但不弹系统通知。

2. **「一眼足够」的信息密度**
   - 浮窗仅展示：名称（最多 5 字）、现价、当日涨跌幅；
   - 通过颜色与存在感层级让用户“余光一扫就知道今天大概什么状态”；
   - 详细信息（第二层）通过悬停或轻交互再呈现，避免信息过载。

3. **情绪温度计而非交易终端**
   - 插件定位不是交易工具，而是“情绪温度计”：
     - 今天大盘情绪如何；
     - 核心票是否有异常波动；
     - 是否值得中午 / 收盘前翻看一次 App / 券商终端。

### 4. 版本规划与迭代路线

#### v1.0（当前基础版）

- 基础行情展示、轮播逻辑、定高循环展示；
- Content Script 浮窗 + Background 代理；
- 自选股管理（添加 / 删除 / 自动补前缀）；
- 上证指数常驻标题栏；
- 无闪烁更新；
- 存在感三态（基础版）与静默态降频刷新；
- README / PRD / DEVELOPMENT 等基础文档。

#### v1.1：上架前打磨（P0）

**目标**：可上架 Chrome Web Store 的“干净稳定版”。

- 功能 / 体验：
  - 在浮窗中提供极简的首次使用提示（例如：仅在默认股票时显示“点击 ⚙️ 添加你关注的股票”），用户添加任意自选后不再展示；
  - 统一网络与错误文案，强调“网络不可用 / 数据可能过时 / 网络重连中…”等办公友好语气；
  - 强化 README 中的权限与隐私说明（仅访问行情接口，不上传用户数据）。
- 技术 / 稳定性：
  - 补齐所有 `chrome.runtime.*` 调用的错误处理（已基本完成）；
  - 针对扩展 reload / context invalidated 做优雅降级，不出现未捕获错误。

#### v1.2：摸鱼友好度增强（P1）

**目标**：让插件在真实上班场景中“更懂打工人”。

- 工作时间 / 非工作时间策略（Work-aware 模式）：
  - 默认在工作日交易时段内保持浮窗；
  - 夜间 / 周末可配置为“深度静默”：不刷新 / 不轮播 / 甚至可选自动隐藏；
  - 设置面板中提供两项简单开关：
    - “工作日自动开启浮窗”（默认开）；
    - “非交易时间自动降低存在感”（默认开）。
- 关键态柔和提醒：
  - 保持当前的边框高亮与存在感提升；
  - 关键股票优先出现在当前 3 行（已实现基础版本，将继续调优策略）；
  - 在浮窗内部增加一次性的、5 秒自动消失的小灰字提示，例如：“有股票涨跌超 3%，建议抽空看一下。”（不做系统通知）。
- 行情“温度条”：
  - 在标题栏指数右侧，以极简小圆点颜色表示整体情绪（绿 / 灰 / 红）；
  - 不增加任何文字，避免噪音，仅作为视觉辅助。

#### v1.3+：中长期差异化（P2）

**目标**：在不牺牲简洁的前提下，提供“进阶打工人”能力。

- 轻量分组（核心持仓 / 观察列表）：
  - 不做复杂分组管理，只提供两个固定分组；
  - 设置面板中通过简单标签切换“核心 / 观察”，标题栏提供一键切换视图。
- 日内节奏提示：
  - 仅在特定时间段（如 9:30–10:00, 14:30–15:00）在浮窗底部用小灰字提示“波动可能偏大，请理性看待”等；
  - 不弹窗、不抢眼。
- 多市场扩展位（如美股）：
  - 在设置中以高级选项形式出现，默认关闭；
  - 需要先验证数据源和延迟，再决定是否默认开放。

### 5. 上架商店配套工作

- 文案与截图：
  - 标题、副标题、核心卖点围绕“低打扰摸鱼看盘”展开；

  - PRD / DEVELOPMENT：持续同步为内部参考；
  - PRESENCE_STATES_DESIGN：存在感三态的设计与实现说明（已存在）。

### 6. 验收标准（针对升级项）

- 行为：
  - 默认静默态下，不影响用户工作注意力；
  - 关键态出现频率合理（不会因为短暂抖动频繁进出关键态）；
  - 工作时间 / 非工作时间策略行为可预测、可配置。
- 体验：
  - 无系统级弹窗，无声音通知；
  - 状态变化有明显但温和的视觉反馈；
  - 错误与异常始终以“温和文案 + 保留旧数据”的方式呈现。
